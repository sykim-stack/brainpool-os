# BRAINPOOL TEAM DIRECTIVE v1.0

**목적**: BRAINPOOL 팀의 철학적 기반과 협업 원칙 정의

---

## 1. 프로젝트의 본질
BRAINPOOL은 단순한 소프트웨어 개발 프로젝트가 아닙니다. 사람의 언어, 대화, 삶을 각 Core에서 **Knowledge(지식)**로 축적하고 이를 연결하여 사람을 이해하는 OS를 만드는 **디지털 생태계 프로젝트**입니다.

## 2. 개발 순서 원칙
모든 구현은 다음 순서를 절대적으로 준수해야 합니다:
1. **철학 (Philosophy)**: 왜 이 기능이 필요한가? 사람을 어떻게 이해하려 하는가?
2. **설계 (Design)**: 단일 Message 구조에 부합하는가?
3. **구현 (Implementation)**: (ctx) => ctx 형식을 따르는가?

## 3. 에이전트 협업 모델
- **총괄 (클로1)**: 방향성 제시 및 최종 승인
- **설계 (클로2)**: 구조적 단순성 유지 및 인터페이스 설계
- **각 모듈 담당 (클로3, 4, 5)**: 담당 영역의 전문적 구현 및 지식 축적
- **관리 (마누스)**: 객관적 데이터 기반의 진행 상황 보고 및 리스크 감지

## 4. 커뮤니케이션 가이드
- 모든 변경 사항은 **문서화**가 선행되어야 합니다.
- 코드보다 **데이터의 흐름(Context)**을 먼저 논의합니다.
- 복잡한 기능을 추가하기보다 **기존 기능의 통합과 단순화**를 우선합니다.
